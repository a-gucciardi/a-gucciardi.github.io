Setting up this site

I wanted an easy way to have a simple yet custom blog, and customizable without knowing much about front-end development.
This is when I found out about HUGO. (link)

Allowing me to deploy a site with a large variety of minimal (or not) themes ready for launch.
And the best way to then use it for my site is to actually deploy the site online. And a popular and easy way is made thanks to github pages. At that point I realised that I could rather use Jekyll, the github default static site builder. At the moment I am sticking to Hugos based on some technical criterias like ‘Oh I found more simple themes there’ I will stick with hugo.

Anyway, what you have to note is that you are going to need 2 git repos to deploy the site. One public and an other private. The good part is that the private one is where you will be able to do all your changes, which will then be affecting your site, without having to touch more than the parts needed in the private directories. This way, if I want to add a new blog page just like this one I juste have to add a single new file. (Markdown or HTML).

Many guides cover the Hugo installation, but I will just list the steps and commands I used : (mac)

  1. brew install hugo hugo version

if you don’t have git (don’t judge) : git –version

go the directory where you want to setup your whole site directory 2. cd dir hugo new site mysite

-screenshot of hugo message -

As advised, it’s time to pick a theme, but first let’s setup the git repos : 3. run hugo go to public/ git init git add . git remote add origin https://github.com/username/username.github.io git commit -m “first commit” git push origin master

  1. Private repo : cd public git init git add . git remote add origin https://github.com/username/private_repo_name git commit -m “first commit” git push origin master

You can then go to each repo accordingly to change the private/public settings. It is obligatory to have the public part pushed in a public git repo.
5. add theme : pick one and do git submodule add https://github.com/Track3/hermit.git themes/hermit

Check on the .gitmodules file on the git repo if you have, this is for github to understand that you have git repos embedded in your main one. [submodule “public”] path = public url = https://github.com/user/user.github.io [submodule “themes/hermit”] path = themes/hermit url = https://github.com/Track3/hermit.git

My site, hugo + github pages.

Setting up this site

I wanted an easy way to have a simple yet custom blog, and customizable without knowing much about front-end development.
This is when I found out about HUGO. (link)

In addition, it can be nicely combined with github pages. At the moment I am sticking to HUGO based on simple personal preference and themes choice, but I realised that I could rather use Jekyll, the github default static site builder.

Note that you are going to need 2 git repos to deploy the site. One public and the other private. The good part is that the private one is where you will be able to do all your desired changes. These updates will then be affecting your site, without having to touch more than the pages. For example, if I want to add a new blog page just like this one I juste have to add a single new file. (Markdown or HTML).

Many guides cover the Hugo installation, but I will just list the steps and commands I used : (macOS version)

Directories setup

  1. brew install hugo
    hugo version

go the directory where you want to setup your whole site directory 2. cd dir hugo new site mysite

-screenshot of hugo message -

I personaly prefer to first setup the git repos :

  1. run hugo go to public/ git init git add . git remote add origin https://github.com/username/username.github.io git commit -m “first commit” git push origin master

  2. Private repo : cd public git init git add . git remote add origin https://github.com/username/private_repo_name git commit -m “first commit” git push origin master

You can then go to each repo accordingly to change the private/public settings. It is obligatory to have the public part pushed in a public git repo.

And then pick a theme:

  1. add theme : pick one and do git submodule add https://github.com/Track3/hermit.git themes/hermit

Check on the .gitmodules file on the git repo if you have, this is for github to understand that you have git repos embedded in your main one. [submodule “public”] path = public url = https://github.com/user/user.github.io [submodule “themes/hermit”] path = themes/hermit url = https://github.com/Track3/hermit.git

Finishing

  1. .nojekyll in public
  2. Publication
    Making use of the github actions and deploy keys

At this point everything is set up, and if you run the cmd hugo server in the general dir you should see your site, to make sure it is generated.
